Project Requirements Table

I decided to treat the Portfolio Project Final Submission page's subordinate list headed "A completed Portfolio Project is a directory that includes:" as a checklist. For ease of grading, I've reproduced that list below, adjoined by links to where the files that satisfy each requirement can be found.

If you've just run (or re-run) the alembic migrations and have a fresh, empty database, the ./load_test_data.py script can be run to fill the database with testing data. It loads data for each table (apart from sales_records) from the ./data/ directory, randomly assigning foreign key column values. The sales_records table is populated with randomly generated sales history data for each book in the `books` table (a time-consuming task).
